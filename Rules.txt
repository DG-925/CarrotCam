Standing Rules — always follow these unless I explicitly say otherwise:
Never push to GitHub or publish a release without my explicit go-ahead. Building/compiling locally is fine anytime, but pushing commits, creating tags, or publishing to the releases repo requires me to say "release it" or equivalent — every time, no exceptions, even if a change seems small or a previous release was recently approved.

Every release build must be fully self-contained — no missing files, ever. Before telling me a build is ready, you must:

List every DLL, dependency, and resource file the app actually needs to run on a clean machine (no Qt, no dev tools installed).
Confirm each one is actually present in the output folder — don't assume, check.
Run windeployqt (or the correct equivalent) and verify its output is complete, not just that it ran.
If you're not 100% sure a file is needed, include it rather than leave it out — missing files are a much bigger problem than a slightly larger folder.
Tell me explicitly, as a checklist, what's included in the build before I test it — don't just say "done."


If a previous release was missing files, treat that as a process failure, not a one-off mistake. Before the next release, explain what you'll do differently to make sure it doesn't happen again (e.g. a checklist you'll run through every time), not just fix the immediate missing file.

After EVERY desktop Release build (clean or incremental), ALWAYS run windeployqt BEFORE telling the user the build is ready. No exceptions. A fresh Release build has zero Qt DLLs — they only appear after windeployqt. If the user reports a missing DLL, it means this step was skipped. The correct sequence is: cmake configure → cmake build → windeployqt → verify DLLs → THEN tell user it's ready.

Every release MUST include all three assets: CarrotCamSetup.exe (Windows installer), CarrotCam-win64.zip (portable), and CarrotCamMobile.apk (Android). No exceptions — all three go into the GitHub release AND into the repo's build/ folder on main.

After every release, always push the build files (CarrotCamSetup.exe, CarrotCam-win64.zip, CarrotCamMobile.apk) to the repo's build/ directory on main so they live alongside the README.md. Commit and push to DG-925/CarrotCam main branch. Build a fresh installer (.exe) every time a new version is released — never reuse an old one.
